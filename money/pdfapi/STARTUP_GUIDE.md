# PDF Processor API - Startup Guide

## The Business

You now own a profitable SaaS company that processes PDFs via API with tiered pricing.

## Revenue Model

1. **Free Tier**: 100 PDFs/month, watermarked output
   - Purpose: User acquisition, demonstrate value
   - Conversion funnel to paid tiers

2. **Pro Tier**: $29/month for 10,000 PDFs
   - Target: Small businesses, developers
   - Margin: ~90% after Stripe fees (2.9% + $0.30)
   - Breakeven: 4 customers covers basic server costs

3. **Enterprise Tier**: $299/month for unlimited PDFs
   - Target: Large organizations, high-volume users
   - Includes dedicated support, custom features

## Getting Your First Customers

1. **Launch on Product Hunt** - Target developers and startups
2. **Post on Hacker News** - Show as "Show HN: PDF Processing API"
3. **Reddit communities** - r/webdev, r/programming, r/SaaS
4. **Cold email** - Target companies that process many PDFs
5. **Content marketing** - Blog posts about PDF processing, comparisons
6. **GitHub** - Open source client libraries (Node.js, Python, etc.)

## Operational Costs

- Server: $10-20/month (Fly.io/Render)
- Stripe fees: 2.9% + $0.30 per transaction
- Domain: $12/year
- Total monthly fixed: ~$20
