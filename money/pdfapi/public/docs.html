API Reference

Everything you need to integrate with the PDF Processor API. Authenticate with your API key and start processing PDFs.

Authentication

All API endpoints (except public ones) require an X-API-Key header containing your Stripe Customer ID.

curl -H "X-API-Key: cus_XXX" https://pdf-processor-api.fly.dev/process-pdf \n  -F "file=@document.pdf"

Base URL

https://pdf-processor-api.fly.dev

All examples use this base URL.

Process PDF

POST /process-pdf

Upload a PDF file for processing (watermarking for free tier, no watermark for paid tiers).

Request: multipart/form-data with field "file"

Response: PDF binary with usage headers.

curl -X POST -H "X-API-Key: cus_XXX" \n  -F "file=@input.pdf" \n  https://pdf-processor-api.fly.dev/process-pdf \n  --output processed.pdf

Customer Usage

GET /customer/usage

Retrieve your current usage, tier, limits, and next billing cycle.

Response JSON:

{
  "customerId": "cus_XXX",
  "tier": "free",
  "usage": 5,
  "limit": 100,
  "remaining": 95,
  "lastReset": "2026-03-18T...",
  "tierPrice": 0,
  "nextBillingCycle": "End of month"
}

Create Checkout Session

POST /create-checkout-session (public)

Generate a Stripe Checkout Session URL to upgrade your plan.

Request JSON: { "priceId": "price_XXX" }

Response: { "sessionId": "...", "url": "..." }

curl -X POST -H "Content-Type: application/json" \n  -d '{"priceId": "price_XXX"}' \n  https://pdf-processor-api.fly.dev/create-checkout-session

Admin Endpoints

Admin endpoints require X-Admin-Token header. Available endpoints:

  • GET /admin/usage – detailed usage across all customers
  • GET /admin/revenue – MRR and customer counts

Webhooks

Stripe webhooks are accepted at POST /webhook for subscription updates.

Configure your Stripe dashboard with signing secret.
